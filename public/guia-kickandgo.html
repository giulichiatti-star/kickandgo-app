2. Ciclo de vida de un cliente

Flujo completo de un entrenador
1
Lead entra — rellena el formulario en la landing → aparece en Admin → Leads con estado Nuevo
2
Contactar — le envías email o WhatsApp → estado pasa a Contactado. Marcas si respondió Sí o No.
3
Dar de alta — seleccionas el lead y pulsas "Dar de alta". Se crea la cuenta, se genera contraseña y se envía email automático con credenciales. Estado → Activo
4
Prueba de 14 días — el entrenador usa la app. A los 11 días recibe aviso por email. Al día 14 recibe email de vencimiento. Tiene 2 días de gracia.
5
Pago — el cliente paga y pulsa "Ya pagué" en el email → aparece en Admin → Avisos de pago. Tú confirmas → cuenta pasa a Pagado
6
Renovación mensual — cada mes el mismo ciclo: email T−1 día, luego 2 días de gracia, luego suspensión.
7
Baja — si decide irse, pulsas "Dar de baja" → pierde acceso. Datos conservados 30 días por RGPD.

3. Panel Admin — Leads

¿Cómo contactar a un lead?

1
Ve a Admin → Leads
2
Filtra por "Nuevo" o pulsa "Seleccionar todos los nuevos"
3
Opción A: "Contactar por email" — envía email automático de bienvenida
Opción B: "Abrir WhatsApp" — abre una pestaña por lead con mensaje ya escrito
4
Marca si el lead "Respondió: Sí / No"
5
Usa el campo de notas para anotar seguimiento, cuándo llamar, etc.

¿Cómo dar de alta a un cliente?

1
Selecciona el lead marcando su checkbox
2
Pulsa "Dar de alta" → confirma en el modal
3
El sistema crea la cuenta, genera contraseña y envía email al cliente con sus credenciales
4
Si el email falló, se muestra la contraseña para enviársela por WhatsApp
⚠️ Si el email de alta no llega, copia la contraseña del resultado y envíasela manualmente por WhatsApp.

4. Panel Admin — Cuentas y cobros

Confirmar un pago recibido

1
Si el cliente pulsó "Ya pagué" → ve a Admin → Avisos de pago (badge rojo con el número)
2
Verifica en tu banco o Bizum que el pago llegó
3
Pulsa "Confirmar pago" → la cuenta se marca como Pagada y el contador avanza un mes

Marcar pago manualmente

1
Ve a Admin → Cuentas → encuentra al cliente
2
Pulsa "💳 Marcar pagado" → la fecha de próximo vencimiento se calcula sola
AcciónCuándo usarla
⚠️ MoraEl cliente avisó que pagará tarde
🚫 Dar de bajaEl cliente quiere cancelar definitivamente
✅ ReactivarCliente suspendido que ya pagó
🔑 Nueva contraseñaEl cliente olvidó su contraseña
🔥 Marcar fundadorPrimeros 50 clientes con precio especial
🗑️ Eliminar cuentaBorrar todos los datos — solo si el cliente lo pide (RGPD)
🗑️ Eliminar cuenta es IRREVERSIBLE. Borra jugadores, partidos, entrenamientos, convocatorias y tarjetas. Solo hacerlo si el cliente lo solicita explícitamente.

Los emails se envían solos cada día a las 9:00h. No tienes que hacer nada.

T+2 (suspensión)Prueba sin pagar"No hemos constatado tu pago…"
T−1 díaClientes de pago"Tu suscripción vence mañana"

Todos los emails incluyen botones "Ya pagué con transferencia" y "Ya pagué con Bizum". Al pulsar, el sistema registra el aviso y aparece en el panel.

Para verificar que los crons están activos: Supabase → SQL Editor → select * from cron.job;

7. Mantenimiento técnico

Qué revisar cada semana

  • Admin → Avisos de pago: confirmar los pendientes
  • Admin → Cuentas → "Alertas de cobro": ver quién está por vencer
  • Sentry (sentry.io): revisar si hay errores nuevos

Qué hacer si un email no llegó

1
Entra en resend.com → Emails → filtra por el email del cliente
2
Si aparece "Bounced" → el email del cliente está mal escrito, corrígelo en Supabase → profiles
3
Si no aparece → verifica los crons: Supabase → SQL Editor → select * from cron.job;

Cómo resetear contraseña de un cliente

1
Admin → Cuentas → busca al cliente → "🔑 Nueva contraseña"
2
Copia la contraseña (solo se muestra una vez)
3
Envíasela por WhatsApp y dile que la cambie desde Ajustes

Cómo hacer deploy de un cambio

1
Terminal en la carpeta /Documents/5_test/kickandgo-app
2
git add -A && git commit -m "descripción" && git push
3
Vercel despliega automáticamente en 1-2 minutos

8. Preguntas frecuentes

Un cliente dice que no puede entrar

  • Verifica en Admin → Cuentas que la cuenta esté activa y no suspendida
  • Si olvidó contraseña → resetéala desde Admin
  • Si está suspendida → confirma el pago o reactiva manualmente

Un cliente quiere cancelar

  • Solo parar: pulsa "Dar de baja" → pierde acceso, datos conservados
  • Quiere borrar datos (RGPD): pulsa "Eliminar cuenta" — irreversible
  • Tiene 14 días de desistimiento desde el primer pago real

El cliente quiere cambiar su email

  • Supabase → Authentication → Users → editar usuario → cambiar email
